## **III. ORGANIC PRICE DISCOVERY**

Instead of pure supply/demand (requires many traders), use **hybrid model:**

### **Intrinsic Value Formula**

Every word has a base **Intrinsic Value (IV)** that updates with events:

```
Intrinsic Value = Base Value + Event Value + Momentum Value
```

**Base Value:** 
- All words start at 1.00 WB
- Floor price (can't go below 0.10 WB)

**Event Value:**
- Each power event adds points
- 1 point = 0.01 WB
- Example: 500-point event adds 5.00 WB to intrinsic value
- Decays 1% per day without new events (prevents infinite growth)

**Momentum Value:**
- Recent events worth more than old ones
- Last 7 days: 100% weight
- 8-30 days: 50% weight  
- 31-90 days: 25% weight
- 90+ days: 0% weight

**Example:**
- "Fuck" base value: 1.00 WB
- Had 750-point event 5 days ago: +7.50 WB
- Had 200-point event 20 days ago: +1.00 WB (50% weight)
- **Current Intrinsic Value: 9.50 WB**

### **Market Price vs Intrinsic Value**

**With 1 User (You Alone):**
- Platform acts as market maker
- You buy/sell at Intrinsic Value Â± 2% (small spread for platform fee)
- Buy price: IV Ã— 1.02
- Sell price: IV Ã— 0.98

**With Multiple Users:**
- Order book appears (users trade with each other)
- Market price can diverge from IV (speculation!)
- If market price > IV: Word is "overvalued" (risky)
- If market price < IV: Word is "undervalued" (opportunity)
- Arbitrage opportunities emerge naturally

**This means:**
- System works perfectly with 1 user (trade with platform)
- Scales organically as users join (trade with each other)
- Price discovery happens naturally without complex algorithms

**Automatic Dividend Payment:**
- Event confirmed â†’ points converted to WB (1 point = 0.01 WB)

## **XIII. ANTI-MANIPULATION & FAIRNESS**

### **Manipulation Vectors & Defenses**

**1. Pump & Dump Schemes**
- *Attack*: Coordinated buy to spike price, then dump
- *Defense*:
  - Max 5% per order (can't whale buy)
  - Wash trading detection
  - Volume spike alerts
  - User reputation system

**2. Insider Trading**
- *Attack*: You know politician will say word tomorrow
- *Defense*:
  - Trading paused before scheduled events
  - Community reporting
  - Pattern detection algorithms

**3. Fake Event Reporting**
- *Attack*: Submit fake news to trigger dividends
- *Defense*:
  - Multi-source verification
  - Human validation layer
  - Trusted domain whitelist
  - Permanent ban for false reports

**4. Front-Running**
- *Attack*: See large order coming, trade ahead
- *Defense*:
  - Order batching (all orders in 5-sec window execute together)
  - Fair sequencing protocol

### **Reputation System**

Users earn reputation via:

```sql
    balance DECIMAL(12,2) DEFAULT 10000.00,
    created_at TIMESTAMP DEFAULT NOW()
);

-- Words
CREATE TABLE words (
    id SERIAL PRIMARY KEY,
    text VARCHAR(100) UNIQUE NOT NULL,
    sector VARCHAR(50),
    base_value DECIMAL(10,2) DEFAULT 1.00,
    intrinsic_value DECIMAL(10,2) DEFAULT 1.00,
    total_shares INTEGER DEFAULT 1000,
    listed_at TIMESTAMP DEFAULT NOW(),
    submitter_id INTEGER REFERENCES users(id)
);

-- Holdings
CREATE TABLE holdings (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    word_id INTEGER REFERENCES words(id),
    quantity INTEGER NOT NULL,
    avg_cost DECIMAL(10,2),
    UNIQUE(user_id, word_id)
);

-- Transactions
CREATE TABLE transactions (
    id SERIAL PRIMARY KEY,
    buyer_id INTEGER REFERENCES users(id),
    seller_id INTEGER REFERENCES users(id),
    word_id INTEGER REFERENCES words(id),
    quantity INTEGER NOT NULL,
    price DECIMAL(10,2) NOT NULL,
    fee DECIMAL(10,2) NOT NULL,
    created_at TIMESTAMP DEFAULT NOW()
);

-- Events
CREATE TABLE events (
    id SERIAL PRIMARY KEY,
    word_id INTEGER REFERENCES words(id),
    points INTEGER NOT NULL,
    description TEXT,
    source_url TEXT,
    validated BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT NOW()
);

-- Dividends
CREATE TABLE dividends (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    word_id INTEGER REFERENCES words(id),
    event_id INTEGER REFERENCES events(id),
    amount DECIMAL(10,2) NOT NULL,
    paid_at TIMESTAMP DEFAULT NOW()
);
```

### **Questions to Answer Before Coding:**

   - Reason: Avoid regulatory complexity

   - **Recommended**: Web first, mobile later
   - Reason: Faster iteration

   - **Recommended**: Hybrid (automated + spot-check)
   - Reason: Balance accuracy and scalability

4. **Initial word pool size?**
   - **Recommended**: 50-100 curated words
   - Reason: Quality over quantity at launch

5. **Blockchain or traditional database?**
   - **Recommended**: Traditional (PostgreSQL)
   - Reason: Faster, cheaper, easier

   - **Recommended**: 3-4 months to MVP
   - Month 1: Core engine
   - Month 2: Basic UI
   - Month 3: Event system
